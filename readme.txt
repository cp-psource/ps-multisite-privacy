Fügt mehr Datenschutz hinzu und ermöglicht es, diese auf allen Websites zu steuern - oder Benutzern zu erlauben, sie zu überschreiben.

Adds more privacy and lets you control it across sites - or allow users to override it.

== Description ==

= DEUTSCH = 

PS Multisite Privatsphäre fügt netzwerkweite Datenschutzstufen hinzu und ermöglicht Dir zu entscheiden, ob Benutzer diese überschreiben können.

Mehr Ebenen, mehr Kontrolle
Mit diesem Plugin hast Du die volle Kontrolle über den Datenschutz in Deinem Netzwerk.

Einfach Netzwerkweit aktivieren, wähle aus den Einstellungen im Netzwerk-Dashboard die für Dich passenden aus. Fertig!

= Kurzanleitung =

= Datenschutzoptionen bei der Webseiten-Registrierung anzeigen =

Die Option "Datenschutz bei der Anmeldung anzeigen" wird angezeigt, wenn entweder "Angemeldete Benutzer können neue Webseiten registrieren" oder "Beides Webseiten und Benutzerkonten können registriert werden" unter Registrierungseinstellungen in **Einstellungen > Netzwerkeinstellungen** im Netzwerkadministrator-Dashboard ausgewählt ist. 
Wenn "Ja" ausgewählt ist, können Benutzer ihre bevorzugten Datenschutzoptionen beim Erstellen ihrer Webseite festlegen.

= Verfügbare Optionen =

Mit den verfügbaren Optionen kann der Super-Admin-Benutzer steuern, welche Datenschutzoptionen unter **Einstellungen > Lesen** im Webseiten-Admin-Dashboard angezeigt werden.

= Standardeinstellungen =

Standardeinstellungen steuern die Datenschutzeinstellungen aller neu erstellten Webseiten. Wenn Du beispielsweise alle neu erstellten Webseiten privat machen möchtest, wähle entweder:

1.  **Erlaube nur eingeloggten Benutzern, alle Blogs zu sehen** - jeder, der ein registrierter Benutzer in Deinem Netzwerk ist und in seinem Konto angemeldet ist, kann die Webseite anzeigen. Ideal, wenn Du Webseiten privat machen möchtest, ohne Benutzer zu einzelnen Webseiten hinzuzufügen.
ODER
2.  **Erlaube nur einem registrierten Nutzer, einen Blog zu sehen, für den er registriert ist**. Jeder, der ein registrierter Nutzer der Website ist und in seinem Konto angemeldet ist, kann die Website anzeigen, dh jeder, der als Benutzer zur Webseite hinzugefügt wurde, wird unter Benutzer > Alle Benutzer aufgelistet. Wird verwendet, wenn Du den Zugriff auf bestimmte Personen beschränken möchtest.
ODER
3.  **Erlaube nur Administratoren eines Blogs, den Blog anzuzeigen, für den sie Administrator sind**. Jeder, der ein registrierter Administrator der Webseite ist und in seinem Konto angemeldet ist, kann die Webseite anzeigen. Ideal für Testzwecke, bevor eine Webseite veröffentlicht wird.

= Überschreiben zulassen =

Überschreiben zulassen lässt Dich entscheiden, ob Blog-Admin-Benutzer die Datenschutzoptionen unter Einstellungen > Lesen ändern können. Du würdest beispielsweise "Nein" auswählen, wenn Du alle Webseiten privat halten möchtest.

= Alle Websites aktualisieren =

Alle Webseiten aktualisieren wurde entwickelt, um die Datenschutzeinstellungen auf allen Webseiten in Deinem Netzwerk zu aktualisieren. Du würdest beispielsweise Alle Webseiten aktualisieren verwenden, wenn Deine Standardeinstellung "Alle Besucher für alle Blogs zulassen" lautet und Du alle Webseiten schnell privat machen musst.

Bitte beachte:

1.  Wähle „Alle Webseiten aktualisieren“, um vorhandene Blogs zu aktualisieren und auf alle neu erstellten Blogs anzuwenden..
2.  Wähle nicht „Alle Blogs aktualisieren“, wenn Du nur möchtest, dass die neuen Datenschutzvorgaben für alle neu erstellten Webseiten gelten..
3.  "Alle Websiten aktualisieren" aktualisiert nicht die Datenschutzeinstellungen der Hauptseite. Dies liegt daran, dass es üblich ist, die Hauptwebsite auf "Alle Besucher für alle Blogs zulassen" festzulegen, damit Du Community-Support und Hilfe auf der Hauptwebseite bereitstellen kannst. Um die Datenschutzeinstellungen der Hauptseite zu ändern, musst Du im Webseiten-Admin-Dashboard der Hauptseite zu Einstellungen > Lesen gehen.

= Ändern der Datenschutzeinstellungen auf einer Webseite-für-Webseite-Basis =

Du kannst den Datenschutz einer Webseite ändern, indem Du: 
1.  beim Webseiten-Admin-Dashboard der Webseite anmeldest 
2. zu **Einstellungen > Lesen** gehst 
3.  Deine neue Datenschutzeinstellung auswählst. 
4.  Klicke auf **Änderungen speichern**.

= ENGLISH =

PS Multisite Privacy adds network-wide privacy levels and allows you to decide whether users can override them.

More levels, more control
With this plugin you have full control over data protection in your network.

Simply activate network-wide, select the settings that suit you from the network dashboard. Complete!

= Quick guide =

= Show privacy options at website registration =

The "Show privacy at login" option is displayed when either "Logged in users can register new websites" or "Both websites and user accounts can be registered" is selected under Registration Settings in **Settings> Network Settings** in the Network Admin Dashboard.
If "Yes" is selected, users can set their preferred privacy options when creating their website.

= Options available =

The options available allow the super admin user to control which privacy options are displayed under **Settings > Reading** in the website admin dashboard.

= Default settings =

Default settings control the privacy settings of all newly created websites. For example, if you want to make all newly created websites private, choose either:

1. **Only allow logged in users to see all blogs** - anyone who is a registered user in your network and is logged into their account can view the website. Ideal if you want to make websites private without adding users to individual websites.
OR
2. **Allow only a registered user to see a blog they are registered for**. Anyone who is a registered user of the site and is logged into their account can view the site, meaning anyone added as a user to the site will be listed under Users > All Users. Used when you want to restrict access to specific people.
OR
3. **Allow only admins of a blog to view the blog they are admins of**. Anyone who is a registered administrator of the website and is logged into their account can view the website. Ideal for testing purposes before a website is published.

= Allow overwriting =

Allow Overwrite lets you decide whether blog admin users can change privacy options under Settings > Reading. For example, you would select "No" if you want to keep all websites private.

= Update all sites =

Update All Websites is designed to update the privacy settings on all websites in your network. For example, you would use Refresh all websites if your default setting is "Allow all visitors for all blogs" and you need to quickly make all websites private.

Please note:

1. Select "Update All Sites" to update existing blogs and apply them to all newly created blogs.
2. Don't choose "Update all blogs" if you just want the new privacy policy to apply to all newly created websites.
3. "Refresh All Sites" does not update the privacy settings of the main page. This is because it's common to set the main website to "Allow all visitors for all blogs" so that you can provide community support and help on the main website. To change the main page privacy settings, you need to go to Settings > Reading in the main page website admin dashboard.

= Changing privacy settings on a site-by-site basis =

You can change a website's privacy policy by:
1. Log into the site's site admin dashboard
2. Go to **Settings > Reading**
3. Select your new privacy setting.
4. Click **Save Changes**.
